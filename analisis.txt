================================================================================

  Frontend: Razor Views + CSS personalizado + JavaScript vanilla
  Autenticacion: Cookies con ClaimsPrincipal (30 min, sliding expiration)

  Proposito: Digitalizar el registro de asistencias del ISFDyT Nro 124.
  Organizado en 4 grupos de trabajo:
    Grupo 1 - Login y recuperacion de contrasena
    Grupo 2 - ABM Usuarios y panel administrativo
    Grupo 3 - ABM Carreras y ABM Materias
    Grupo 4 - Asistencias y Asistencia Global

3. LO QUE ESTA INCOMPLETO O CON ERRORES
================================================================================

  3.1. ASISTENCIAS (Grupo 4) - PRIORIDAD ALTA
  -------------------------------------------
  - AsistenciasController.cs: 100% comentado. Ninguna accion funcional.
  - Views/Home/Asistencia.cshtml: maqueta estatica con datos hardcodeados.
    Los checkboxes de modulos y justificacion no se conectan a la DB.
    El boton "Guardar" solo muestra un alert() y redirige sin persistir.
  - Views/Home/AsistenciaGlobal.cshtml: maqueta estatica con porcentajes
    hardcodeados. Sin calculos reales desde la DB.
  - Sin logica de negocio para tomar asistencia (presente/ausente/justificado).

  3.2. SEGURIDAD - CONTRASENAS - PRIORIDAD ALTA
  ---------------------------------------------
  - Las contrasenas se almacenan en TEXTO PLANO (sin hashing).
  - Login compara strings directamente: usuario.UsContrasena != model.Contrasena
  - Seed data: UsContrasena = "12345678"
  - Al crear usuario: UsContrasena = model.UsDni.ToString()
  - No hay hashing (ni BCrypt, ni PBKDF2, ni ASP.NET Identity PasswordHasher).
  - No hay politica de complejidad de contrasenas.

  3.3. IDs AUTO-INCREMENTALES - PRIORIDAD ALTA
  --------------------------------------------
  - DbContext usa ValueGeneratedNever() en TODAS las entidades.
  - NO se usa IDENTITY en SQL Server para las PKs.
  - AdminController asigna UsId manualmente (Max(UsId) + 1) -> OK.
  - MateriasController.Create() NO asigna MaId -> CRASH asegurado.
  - CarrerasController.Create() NO asigna CaId -> CRASH asegurado.
  - El modelo acepta CaId desde el formulario via [Bind], pero la vista
    no tiene campo para CaId -> el valor default es 0 -> viola PK.

  3.4. MATERIAS CONTROLLER - PRIORIDAD MEDIA
  ------------------------------------------
  - No tiene [Authorize(Roles = "Admin")]. Cualquier usuario autenticado
    puede acceder al CRUD de materias.
  - Las vistas de Materias usan clases Bootstrap default (form-control,
    btn btn-primary, table) que NO coinciden con el diseno personalizado
    del sistema (clases custom como custom-select, btn-guardar, crud-table).

  3.5. PAGINA DE ACCESO DENEGADO - PRIORIDAD ALTA
  -----------------------------------------------
  - Program.cs configura: AccessDeniedPath = "/Home/Privacy"
  - Home/Privacy.cshtml NO EXISTE -> causa error 404 cuando un usuario
    sin permisos accede a una pagina restringida.

  3.6. CAMBIO DE CONTRASENA - PRIORIDAD MEDIA
  -------------------------------------------
  - No se verifica la contrasena actual antes de cambiarla.
  - No se valida que la nueva contrasena sea diferente a la anterior.
  - Las vistas de Login y CambiarContrasena NO usan el layout principal
    y NO incluyen validacion del lado cliente (jQuery Validation).

  3.7. MENSAJES DE EXITO/ERROR - PRIORIDAD MEDIA
  ----------------------------------------------
  - No hay TempData["Success"] ni TempData["Error"] en ningun controller.
  - No hay notificaciones visuales (toast, alert, banner) tras operaciones
    exitosas.
  - Solo se usa alert() de JavaScript en algunas vistas.

  3.8. CARRERAS CONTROLLER - PRIORIDAD BAJA
  -----------------------------------------
  - Usa el modelo directamente con [Bind] en lugar de DTOs.
  - [Bind] expone propiedades de navegacion (CarrerasCohortes,
    CarrerasMaterias) a overposting.
  - No hay validacion de unicidad en denominacion.

  3.9. ERRORES EN ARCHIVOS ESTATICOS - PRIORIDAD BAJA
  ----------------------------------------------------
  - wwwroot/css/style.css: regla ::-webkit-scrollbar-thumb anidada
    incorrectamente dentro de ::-webkit-scrollbar-track (llaves mal cerradas).
  - wwwroot/js/script.js: intenta cargar datos de usuarios desde
    localStorage con IDs que NO coinciden con los nombres de campo reales
    del formulario (ej: busca "apellidos" pero el input es "UsApellido").

6. ARQUITECTURA TECNICA
================================================================================

  Lenguaje: C# 12 (.NET 8)
  Framework: ASP.NET Core 8.0 MVC
  ORM: Entity Framework Core 8.0.27
  BD: SQL Server (localhost:1433, Docker)
  Autenticacion: Cookies con ClaimsPrincipal
  Frontend: Razor Views + CSS + JavaScript vanilla + jQuery

  Estructura de directorios:
  -------------------------
  ISFDyT124/
    |- Controllers/   (6 controladores)
    |- Models/        (10 modelos de dominio)
    |- DTO/           (20 objetos de transferencia de datos)
    |- Views/         (vistas Razor por controlador)
    |- Data/          (DbContext con Fluent API)
    |- Migrations/    (2 migraciones + snapshot)
    |- wwwroot/       (css, js, images, lib)
    |- Properties/    (launchSettings)
    |- Program.cs     (punto de entrada y configuracion)

7. MODELO DE DATOS
================================================================================

  Tablas (9):

  Roles (RoId PK, RoDenominacion)
  Usuarios (UsId PK, UsApellido, UsNombre, UsDni UNIQUE, UsEmail,
            UsContrasena, RoId FK->Roles, CaCoId FK->CarreraCohorte nullable)
  UsuarioRoles (UsRoId PK, UsId FK->Usuarios, RoId FK->Roles)
  Carreras (CaId PK, CaDenominacion)
  Cohortes (CoId PK, CoAnio)
  Materias (MaId PK, MaDenominacion, MaModalidad, MaCantModulos)
  CarreraCohortes (CaCoId PK, CaId FK->Carreras, CoId FK->Cohortes)
  CarreraMaterias (CaMaId PK, CaId FK->Carreras, MaId FK->Materias)
  Asistencias (AsId PK, AsFecha, AsPresente, AsJustificacion,
               UsId FK->Usuarios, MaId FK->Materias)
  UsuarioCarreraMateria (tabla intermedia M:N Usuario <-> CarreraMateria)

  Nota: Ninguna PK usa auto-incremento IDENTITY. Se asignan manualmente.
